# FLICKER: Searching for Exoplanets with ML

### Some background?

When I was little, I dreamed of someday being an astronaut; While my dreams have changed in the years since, my passion for outer space has stayed with me for as long as I can remember. I’ve been constantly interested in deep space and the missions and science behind them; While I have never done anything overly formal for the subject, I have always followed along and pushed myself to learn more about it in my own time. In doing so, I learned an enthusiasm for the physics that describe our cosmos, as well as the computer science that allows for the analysis of the massive datasets modern astronomy requires. Going into university, I plan to study computer science, with a smaller focus on astrophysics and related subjects where possible. Many of the skills that I will use or learn for this project will be incredibly useful going forward into post-secondary schooling and later into any potential career options.

Therefore, my research question becomes as follows:

#### How can a machine learning (ML) network learn to identify exoplanet candidates using data from the Transiting Exoplanet Survey Satellite (TESS) observatory, and do so on consumer hardware?

From the ML perspective, this is a great entry into how many of the current large language models (LLMs) are designed; knowing why and how decisions are made with machine learning is going to be incredibly valuable going forward in university. Furthermore, exoplanets are a relatively new focus in astronomy, where large datasets like TESS make it possible for almost anyone to make meaningful contributions to the field without the need for large government funding and years of specific research.

The general plan for the capstone project may be defined in 5 steps: Data acquisition, preprocessing, model building, candidate generation, and the final presentation product. First, I plan to compile a large set of labelled light curves — in addition to confirmed false positives — from MAST, the NASA Exoplanet Archive, and ExoFOP-TESS. Second, I will have to detrend and normalize the raw curves to remove instrumental systematics and stellar variability without distorting any potential transit signals. Third, I will design and train a neural network on the dataset, using a validation split for tuning and held-out confirmed cases for evaluation. Fourth, I will run the trained model on unclassified curves and then examine flagged signals manually for their plausibility based on transit shape, periodicity, host star contamination, and other factors. Finally, I will package the trained model and results together into a finalized set of flagged candidates for my presentation.

### Regarding the acronym

FLICKER stands for "Fast Light-curve Inspection and Classification for Keplerian Exoplanet Recognition"  
Don't judge it, NASA does them too.
